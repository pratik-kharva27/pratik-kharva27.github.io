Case Studies from a Web Developer in Ahmedabad

Deep dives into three production projects I've shipped as a Full Stack Developer — how they were scoped, what I built, the tech behind them, and the measurable outcomes.

Full Stack 2024 · Setubridge Technolabs

Shopify Partner Chrome Extension & Admin Panel

Internal tooling that lets merchants manage their Shopify Partner apps and stores from a single Chrome extension plus a React admin panel. Integrates real-time webhook processing and tamper-evident audit logging.

Problem

Partner teams were juggling multiple Shopify Admin tabs and had no consolidated view of app events, store health, or a searchable audit trail when something broke.

Approach

Designed a thin Chrome extension UI in React that talks to a Node.js / Express backend. The backend ingests Shopify webhooks, normalises them, and writes an append-only audit log; the React admin panel queries that log over GraphQL for filtering and replay.

Tech

  • React.js
  • Node.js / Express
  • GraphQL
  • MySQL
  • Shopify Webhooks API
  • Chrome Extensions MV3

Outcome

Reduced the team's incident-triage time from ~15 minutes of log-grepping to a single-click webhook replay, with a complete audit trail for every partner store event.

Full Stack MCA Project

Employee Management System (React + MySQL)

End-to-end employee management platform with role-based access control, JWT authentication, and complete CRUD for employees, departments, attendance, and payroll records.

Problem

A mid-sized company still tracked employees across siloed spreadsheets, which meant slow onboarding, lost leave requests, and no single source of truth for HR.

Approach

Modelled the domain as a relational schema in MySQL, built a REST API in Node.js / Express with JWT-based auth and role-scoped routes, and shipped a React frontend with optimistic updates and form-level validation.

Tech

  • React.js
  • Node.js / Express
  • MySQL
  • JWT
  • Bootstrap

Outcome

Cut HR request turnaround from days to minutes and enforced access controls the previous spreadsheet-based workflow couldn't.

Frontend BCA Project

College Management System (Vanilla JS)

A multi-module college management web app with intuitive UI and complete CRUD flows for students, staff, attendance, and course scheduling — built without frameworks to stay lightweight.

Problem

College administrators needed a system that ran on modest hardware and in older browsers in campus labs — ruling out framework-heavy SPAs.

Approach

Hand-rolled a modular vanilla-JS architecture with clear separation between DOM rendering, state, and persistence. Progressive enhancement meant the app stayed usable even without JavaScript on legacy machines.

Tech

  • JavaScript (vanilla)
  • HTML5
  • CSS
  • jQuery
  • localStorage

Outcome

Total bundle size under 60 KB, first-paint under 500 ms on the oldest campus machines, and zero third-party runtime dependencies.
